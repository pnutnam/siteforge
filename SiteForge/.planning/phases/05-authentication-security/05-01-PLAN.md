  <action>
    Create src/auth/jwt.ts implementing JWT signing and verification using `jose` library with HS256 algorithm.

    **JWT_SECRET handling:**
    - Read from process.env.JWT_SECRET
    - If not set, generate crypto.randomBytes(64) and log warning (for dev only)
    - Require JWT_SECRET >= 32 bytes in production

    **createAccessToken(payload: AccessTokenPayload): Promise<string>**
    - Use SignJWT from jose
    - Set protected header: { alg: 'HS256' }
    - Set expiration: '15m' (15 minutes)
    - Sign with JWT_SECRET

    **verifyAccessToken(token: string): Promise<AccessTokenPayload | null>**
    - Use jwtVerify from jose
    - Verify with algorithms: ['HS256']
    - Return payload as AccessTokenPayload or null if invalid/expired

    **decodeAccessToken(token: string): AccessTokenPayload | null**
    - Use decodeJwt from jose (no verification, for logging)
    - Return payload or null

    ```typescript
    import { SignJWT, jwtVerify, decodeJwt } from 'jose';

    const getJwtSecret = () => {
      const secret = process.env.JWT_SECRET;
      if (!secret) {
        console.warn('JWT_SECRET not set, generating temporary secret');
        return crypto.randomBytes(64);
      }
      if (secret.length < 32 && process.env.NODE_ENV === 'production') {
        throw new Error('JWT_SECRET must be at least 32 bytes');
      }
      return new TextEncoder().encode(secret);
    };

    const JWT_SECRET = getJwtSecret();

    export async function createAccessToken(payload: AccessTokenPayload): Promise<string> {
      return new SignJWT({ ...payload })
        .setProtectedHeader({ alg: 'HS256' })
        .setIssuedAt()
        .setExpirationTime('15m')
        .sign(JWT_SECRET);
    }

    export async function verifyAccessToken(token: string): Promise<AccessTokenPayload | null> {
      try {
        const { payload } = await jwtVerify(token, JWT_SECRET, { algorithms: ['HS256'] });
        return payload as unknown as AccessTokenPayload;
      } catch {
        return null;
      }
    }

    export function decodeAccessToken(token: string): AccessTokenPayload | null {
      try {
        const payload = decodeJwt(token);
        return payload as unknown as AccessTokenPayload;
      } catch {
        return null;
      }
    }
    ```

  <action>
    Create src/auth/refresh.ts implementing refresh token handling with Redis storage.

    **Refresh token approach:** Opaque random token stored in Redis (key = SHA256 hash of token, value = RefreshTokenData + expiry). This enables revocation while keeping refresh token format simple.

    **Token format:** crypto.randomBytes(32).toString('hex') (64 character hex string)

    **createRefreshToken(accountId: string, tenantId: string): Promise<string>**
    - Generate opaque token: crypto.randomBytes(32).toString('hex')
    - Store in Redis: key = `refresh:${sha256(token)}`, value = JSON.stringify({ accountId, tenantId, version: 1 }), expiry = 30 days
    - Return the raw token (hashed version stored in Redis)

    **verifyRefreshToken(token: string): Promise<RefreshTokenData | null>**
    - Hash the token: sha256(token)
    - Look up in Redis: GET refresh:${hash}
    - If found, return parsed RefreshTokenData
    - If not found, return null (token invalid or expired)

    **invalidateRefreshToken(token: string): Promise<void>**
    - Hash the token
    - Delete from Redis: DEL refresh:${hash}

    **Cookie settings:**
    - Cookie name: 'sf_refresh'
    - httpOnly: true
    - secure: true (production)
    - sameSite: 'lax'
    - maxAge: 30 days in seconds (2592000)
    - path: '/'

    Import Redis from ioredis (same connection as BullMQ).
  </action>

    - Refresh token stored in Redis with 30-day expiry
