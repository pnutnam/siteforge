    **Settings (per user decisions):**
    - Window: 5 minutes (300000 ms)
    - Max attempts: 3 per account
    - Key prefix: 'ratelimit:totp:'

    **checkTotpRateLimit(accountId: string): Promise<RateLimitResult>**
    ```typescript
    interface RateLimitResult {
      allowed: boolean;
      remaining: number;
      retryAfter?: number;  // seconds until next allowed attempt
      limit: number;
      reset?: number;       // Unix timestamp when window resets
    }
    ```

    Implementation using Redis sorted sets (ZADD with timestamp score):
    1. Key = `ratelimit:totp:${accountId}`
    2. Remove entries older than window: ZREMRANGEBYSCORE key 0 windowStart
    3. Count current entries: ZCARD key
    4. If count >= max, calculate retryAfter and return { allowed: false }
    5. If allowed, ZADD current timestamp, set expiry to window duration
    6. Return { allowed: true, remaining: max - count - 1 }

    **getRateLimitHeaders(result: RateLimitResult): Record<string, string>**
    Returns headers for 429 responses:
    - X-RateLimit-Limit: '3'
    - X-RateLimit-Remaining: String(result.remaining)
    - X-RateLimit-Reset: String(result.reset)
    - Retry-After: String(result.retryAfter) (only on 429)

    Use the existing Redis connection from BullMQ (ioredis instance exported from jobs/queue.ts).
    Import redis from '@/jobs/queue' (the exported redisConfig or create a separate connection).

    **IMPORTANT:** Use Redis MULTI/EXEC for atomic operations to prevent race conditions.

    ```typescript
    import { redis } from '@/jobs/queue';  // or create ioredis instance

    const RATE_LIMIT_WINDOW_MS = 5 * 60 * 1000;  // 5 minutes
    const RATE_LIMIT_MAX = 3;

    export interface RateLimitResult {
      allowed: boolean;
      remaining: number;
      retryAfter?: number;
      limit: number;
      reset?: number;
    }

    export async function checkTotpRateLimit(accountId: string): Promise<RateLimitResult> {
      const key = `ratelimit:totp:${accountId}`;
      const now = Date.now();
      const windowStart = now - RATE_LIMIT_WINDOW_MS;

      // Atomic: remove old entries + count current + add new entry
      const multi = redis.multi();
      multi.zremrangebyscore(key, 0, windowStart);
      multi.zcard(key);
      multi.zadd(key, now, `${now}:${Math.random()}`);
      multi.expire(key, Math.ceil(RATE_LIMIT_WINDOW_MS / 1000));

      const results = await multi.exec();
      const currentCount = results![1][1] as number;

      if (currentCount >= RATE_LIMIT_MAX) {
        const oldest = await redis.zrange(key, 0, 0, 'WITHSCORES');
        const oldestTime = Number(oldest[1]);
        const retryAfterMs = oldestTime + RATE_LIMIT_WINDOW_MS - now;
        const reset = Math.ceil((oldestTime + RATE_LIMIT_WINDOW_MS) / 1000);

        return {
          allowed: false,
          remaining: 0,
          retryAfter: Math.ceil(retryAfterMs / 1000),
          limit: RATE_LIMIT_MAX,
          reset,
        };
      }

      return {
        allowed: true,
        remaining: RATE_LIMIT_MAX - currentCount - 1,
        limit: RATE_LIMIT_MAX,
        reset: Math.ceil((now + RATE_LIMIT_WINDOW_MS) / 1000),
      };
    }

    export function getRateLimitHeaders(result: RateLimitResult): Record<string, string> {
      const headers: Record<string, string> = {
        'X-RateLimit-Limit': String(result.limit),
        'X-RateLimit-Remaining': String(result.remaining),
        'X-RateLimit-Reset': String(result.reset || Math.ceil(Date.now() / 1000) + 300),
      };
      if (!result.allowed && result.retryAfter) {
        headers['Retry-After'] = String(result.retryAfter);
      }
      return headers;
    }
    ```

    ```typescript
    export async function clearTotpRateLimit(accountId: string): Promise<void> {
      await redis.del(`ratelimit:totp:${accountId}`);
    }
    ```

    Create src/app/api/auth/refresh/route.ts for token refresh.

    **POST /api/auth/refresh**

    Request: No body needed (refresh token comes from httpOnly cookie)

    Response (200):
    ```json
    {
      "token": "eyJhbGciOiJIUzI1NiJ9..."
    }
    ```
    Response sets httpOnly cookie with new refresh token.

    Flow:
    1. Read refresh token from cookie 'sf_refresh'
    2. If no cookie, return 401
    3. Verify refresh token using verifyRefreshToken (checks Redis)
    4. If invalid/expired, return 401 with { error: 'Invalid or expired refresh token' }
    5. Look up account details (email, businessId, tenantId, status) from owner_accounts table
    6. If account status is not 'active', return 403 with { error: 'Account not active' }
    7. Create new access token using createAccessToken with claims from account
    8. Optionally create new refresh token (no rotation - reuse allowed, but refresh the expiry)
    9. Return { token: newAccessToken } with the new access token in the response body
    10. Set new refresh cookie with updated expiry

    **IMPORTANT:** Per user decisions, refresh tokens are NOT rotated on use. They are reused until natural 30-day expiry.

    Cookie settings for refresh:
    ```typescript
    {
      name: 'sf_refresh',
      httpOnly: true,
      secure: process.env.NODE_ENV === 'production',
      sameSite: 'lax',
      maxAge: 30 * 24 * 60 * 60,  // 30 days in seconds
      path: '/',
    }
    ```

    **checkAccountOwnership(accountId: string, tenantId: string): Promise<boolean>**
    Verifies that the account belongs to the specified tenant:
    ```sql
    SELECT tenant_id FROM owner_accounts WHERE id = $1
    ```
    Returns true if tenant_id matches, false otherwise.

    **checkResourceOwnership(accountId: string, tenantId: string, resourceTenantId: string): boolean**
    Simple synchronous check that resource's tenantId matches JWT's tenantId. For when you already have the resource's tenantId.

    **requireOwnership(accountId: string, tenantId: string, resourceTenantId: string): void**
    Throws Error with 'Forbidden' message if ownership check fails.

    ```typescript

    requireOwnership(payload.accountId, payload.tenantId, resource.tenantId);
    ```
